# 면접 종합 평가 보고서

## ✅ Good (장점)
1. **프론트엔드 핵심 경험의 진실성과 일관성**  
   - React 기반 SPA 전환, 컴포넌트화, 상태관리, 성능 개선 등 답변 1의 내용이 자기소개서·포트폴리오와 매우 잘 맞으며, 기술·경력 과장 정황이 거의 없고 실제 수행 경험일 가능성이 높습니다.

2. **모바일 UX 이해와 경험의 일관성**  
   - 모바일 vs 웹 UX 차이, 네트워크 특성에 따른 오프라인 고려 등 답변 2의 내용이 React Native 앱 개발 경력과 자연스럽게 연결되며, 모바일 환경에 대한 기본적인 이해와 실무 경험이 확인됩니다.

3. **사용자 경험·성능에 대한 관심**  
   - 멀티페이지 → SPA 전환, 페이지 전환 속도 향상, 서버 요청 감소, 모바일 환경에서의 로딩·오프라인 고려 등 전반적으로 사용자 경험과 성능을 개선하려는 문제의식이 꾸준히 드러납니다.

4. **문서 기반 기술 스택의 신뢰도**  
   - 자기소개서/포트폴리오에 기재된 React, React Native, 성능 최적화, 크래시 모니터링(Crashlytics/Sentry 등) 관련 경험이 답변 1·2와 방향성·수준이 일치해, 기술 스택 자체의 신뢰도는 비교적 높은 편입니다.

5. **초기 면접 구간에서의 상대적으로 안정된 태도**  
   - 질문 1 구간에서는 Positive 비율이 더 높게 나타나, 초반에는 비교적 안정적이고 호의적인 태도를 보일 수 있는 기반이 있으며, 완전히 일관되게 부정적인 지원자는 아닌 것으로 판단됩니다.

## ❌ Bad (단점)
1. **질문-답변 연관성의 심각한 부족**  
   - 세 질문 모두에서 질문의 핵심 의도(백엔드 지원 동기, 대규모 메시징 시스템 설계, 핵심 API 안정성·모니터링)에 거의 답하지 못했고, 특히 질문 2·3은 백엔드 아키텍처/모니터링과 완전히 동떨어진 내용 또는 욕설로 구성되어 면접의 기본 요건을 충족하지 못했습니다.

2. **백엔드 역량·이해 부족이 드러나는 답변 구성**  
   - Node.js·Express·NestJS, REST API, WebSocket, DB 설계, 캐시, 메시지 큐, 모니터링 지표 등 백엔드 핵심 개념을 질문에서 명시적으로 유도했음에도, 관련 언급이 거의 없거나 전무하여 백엔드 직무 지원자로서의 준비도와 기술 깊이가 크게 의심됩니다.

3. **면접 태도에서의 부정 정서 비중 및 감정 기복**  
   - 전체 감정 분석에서 부정 정서가 60% 이상으로 높고, 특히 질문 2·3에서 disgust·anger 비율이 크게 상승하는 패턴을 보여, 압박·불편한 질문 상황에서 방어적·거부적·짜증 섞인 인상으로 비칠 위험이 큽니다.

4. **전문성에 맞지 않는 언어 사용과 프로페셔널리즘 결여**  
   - 답변 3에서 심한 욕설과 비논리적인 발화가 나타나, 문서에서 강조한 “협업·커뮤니케이션을 중시하는 태도”와 정면으로 충돌하며, 기본적인 직장인·엔지니어로서의 프로페셔널리즘을 의심하게 만드는 수준입니다.

5. **문서와 실제 태도 간 인성·태도 일관성 붕괴**  
   - 자기소개서에는 협업 지향, 소통 중시, 즐겁게 일하는 동료상을 강조했으나, 면접 중(특히 답변 3) 언행은 공격적·감정적·무례하게 나타나 인성·태도 측면의 신뢰도가 크게 떨어지며, 조직 적합성에 대한 강한 리스크 신호로 작용합니다.

## 📈 Development (발전시켜야할 점)
1. **질문 의도 파악 및 구조화된 답변 연습**  
   - 질문에서 요구하는 키워드(예: “백엔드 지원 동기”, “대규모 트래픽”, “모니터링 지표”)를 먼저 머릿속에 정리한 뒤, 상황–과제–행동–결과(STAR) 구조로 답하는 연습을 통해, 질문과 직접 연결된 답변을 짧고 명확하게 구성하는 능력을 키워야 합니다.

2. **백엔드 기본기 및 시스템 설계 역량 강화**  
   - Node.js·Express·NestJS 기반의 REST API 설계, 인증/인가, DB 모델링, 캐시(Redis), 메시지 큐(Kafka 등), 로드밸런싱, 모니터링/알람(Prometheus, Sentry 등)에 대한 기초를 체계적으로 학습하고, 간단한 개인 프로젝트 수준이라도 설계–구현–배포까지 경험을 쌓아야 합니다.

3. **면접 태도·표정 관리 및 감정 조절 능력 개선**  
   - 혐오·분노 계열 표정이 자주 포착되는 만큼, 불편한 질문에서도 중립·가벼운 미소를 기본값으로 유지하는 연습(호흡 조절, 거울·영상 피드백, 모의 면접)을 통해 감정이 얼굴에 직접적으로 드러나지 않도록 관리하는 능력을 길러야 합니다.

4. **전문 환경에 적합한 언어 사용과 커뮤니케이션 훈련**  
   - 욕설·비속어는 어떤 상황에서도 면접·업무 커뮤니케이션에서 사용하지 않도록 철저히 금지하고, 모르는 질문이나 어려운 질문에는 “제가 알고 있는 범위에서 말씀드리면…”, “이 부분은 경험이 부족하지만…”과 같이 차분하고 성숙한 표현으로 대응하는 습관을 들여야 합니다.

5. **문서-행동 간 신뢰도 회복 전략 수립**  
   - 자기소개서에서 주장한 협업 태도·소통 방식이 실제 행동으로도 일치하도록, 팀 프로젝트·사이드 프로젝트에서 피드백을 적극적으로 받고, 갈등 상황·압박 상황에서도 감정을 조절하며 존중 기반으로 대화하는 연습을 통해 인성·태도 측면의 일관성과 신뢰도를 회복할 필요가 있습니다.
