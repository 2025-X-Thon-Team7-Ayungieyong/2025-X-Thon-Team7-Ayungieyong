자소서 예시
어릴 때부터 컴퓨터를 좋아했지만, 단순히 게임을 하는 것보다 “이 화면 뒤에서는 무슨 일이 
벌어질까?”를 더 궁금해하는 타입이었습니다. 대학교에서 컴퓨터공학을 전공하면서 웹과 모
바일 개발 수업을 듣게 되었고, 사용자가 직접 눈으로 보고 손으로 만지는 제품을 만든다는 
점에 매력을 느껴 웹/앱 개발자를 진로로 정하게 되었습니다.
학부 시절에는 HTML/CSS, JavaScript를 기반으로 한 간단한 웹사이트를 만들면서 프론
트엔드 개발의 기본을 익혔고, 이후 React와 TypeScript를 활용해 팀 프로젝트를 진행하며 
컴포넌트 기반 설계와 상태 관리에 대한 이해를 높였습니다. 동시에 서버 사이드에도 관심이 
생겨 Node.js와 Express로 간단한 REST API를 구현하고, PostgreSQL과 MongoDB를 
사용해 데이터를 다루는 경험을 쌓았습니다. 이 과정에서 “기능이 돌아가는 것”을 넘어서 
“유지보수하기 좋은 구조”를 고민하는 습관이 생겼습니다.
졸업 후 첫 직장에서는 약 2년 동안 스타트업에서 웹 프론트엔드 개발자로 일했습니다. 이 회
사에서는 기존에 jQuery 중심으로 되어 있던 레거시 페이지를 React 기반 SPA로 점진적으
로 마이그레이션하는 업무를 담당했습니다. 공통 컴포넌트 라이브러리를 정리하고, 디자인 
시스템에 맞춰 버튼, 입력 폼, 모달 등을 재사용 가능한 형태로 정리하면서 코드 중복을 줄이
고 개발 속도를 높이는 데 기여했습니다. 특히, 마케팅 페이지의 로딩 속도를 개선해 이탈률
을 줄인 경험이 기억에 남습니다. 이미지 최적화, 코드 스플리팅, 불필요한 라이브러리 제거 
등을 통해 LCP 지표를 개선했고, 이를 바탕으로 팀 내에서 성능 개선 가이드를 문서화하기
도 했습니다.
이후에는 약 1년간 모바일 앱 개발이 중심인 회사로 이직해 React Native 기반의 하이브리
드 앱을 개발했습니다. 기존에 웹에서만 제공되던 서비스를 모바일 앱으로 확장하는 프로젝
트에 참여했으며, 로그인/회원가입 플로우, 푸시 알림, 간단한 채팅 화면 등을 구현했습니다. 
웹과 앱의 UX 차이를 직접 경험하며, 같은 기능이라도 모바일 환경에서는 더 단순하고 명확
해야 한다는 점을 몸으로 느꼈습니다. 또한 iOS/Android 스토어 배포 과정, 버전 관리, 크래
시 리포트 확인 등 서비스 운영 단계에서 필요한 실무적인 부분도 함께 경험할 수 있었습니
다.
저는 협업과 커뮤니케이션을 중요하게 생각합니다. 개발자라는 역할이 결국 혼자 코드만 보
는 일이 아니라, 기획자·디자이너·QA와 함께 하나의 제품을 만들어가는 과정이라고 믿기 때
문입니다. 실제로 이전 회사에서는 매주 한 번씩 기능별로 간단한 데모 시간을 갖고, 새로 개
발한 화면을 직접 보여주며 피드백을 바로 받는 자리를 제안하고 진행했습니다. 이 과정에서 
기획 의도를 더 명확히 이해할 수 있었고, 디자이너와도 픽셀 단위가 아닌 사용자 플로우 관
점에서 대화를 나눌 수 있었습니다. 이런 경험을 통해 단순히 “시킨 기능만 구현하는 개발
자”가 아니라 “제품을 함께 만드는 동료”로서의 태도를 꾸준히 키워가고 있습니다.

성격적으로는 꼼꼼한 편이라 작은 버그나 UI 어색함이 눈에 잘 들어오는 편입니다. 그래서 
개발을 마친 뒤에도 단순히 기능 테스트만 하는 것이 아니라, 실제 사용자의 입장에서 여러 
경로로 화면을 돌아다니며 엣지 케이스를 확인하려고 노력합니다. 다만, 이런 꼼꼼함 때문에 
가끔은 사소한 부분에 시간을 너무 많이 쓰는 단점도 있어, 중요도와 우선순위를 먼저 정리
한 뒤 일정 안에서 움직이려고 의식적으로 조절하고 있습니다.
기술적으로는 현재 React, TypeScript, React Query, Next.js 등을 주로 사용하고 있으
며, 간단한 서버 작업은 Node.js(Express, NestJS 기초)를 활용할 수 있습니다. 모바일 쪽
은 React Native를 기반으로 화면 개발과 API 연동, 기본적인 네이티브 모듈 연동 경험을 
가지고 있습니다. 아직 완벽하다고 생각하지 않기 때문에, 개인 시간에는 사이드 프로젝트를 
진행하면서 새로운 라이브러리나 패턴을 적용해 보고, 공식 문서와 기술 블로그를 통해 꾸준
히 공부하고 있습니다.
앞으로의 목표는 “코드를 잘 짜는 개발자”를 넘어서, 사용자의 문제를 이해하고 해결할 수 있
는 웹/앱 개발자로 성장하는 것입니다. 단기적으로는 기존 서비스의 성능과 UX를 안정적으
로 개선하고, 중장기적으로는 기획 단계부터 적극적으로 참여해 제품 방향성에 기여할 수 있
는 개발자가 되고 싶습니다. 또한, 제가 배운 것들을 정리해 팀 내 위키나 기술 공유 세션을 
통해 다른 동료들과 나누며, 함께 성장하는 문화를 만드는 데 기여하고 싶습니다.
귀사에서 웹/앱 개발자로 기회를 주신다면, 지금까지 쌓아 온 웹 프론트엔드 및 모바일 앱 개
발 경험을 바탕으로 사용자에게 편리하고 안정적인 서비스를 제공하는 데 기여하겠습니다. 
맡은 일에 책임감을 가지고 끝까지 파고드는 태도와, 동료와 함께 즐겁게 일하려는 마음가짐
으로 팀에 도움이 되는 구성원이 되겠습니다. 감사합니다.
