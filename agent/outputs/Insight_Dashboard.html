Section 01
Debate Report · 면접 최종 요약
강점·리스크·개선 방향을 5문장 내외로 압축
▶

React SPA 마이그레이션, 성능 개선, 웹·모바일 경험 등 핵심 기술 스택은 문서·답변 전반에 일관되게 드러나 실무 기본 역량과 진정성은 인정됩니다. 반면, “가장 어려웠던 문제” 등 핵심 질문에 대해 구체 사례와 문제–접근–해결–성과 구조가 거의 없어 답변의 논리성과 설득력이 크게 떨어집니다. 특히 욕설·비속어 사용과 높은 부정 정서(anger/disgust) 비율은 프로페셔널리즘과 매너 측면에서 치명적인 탈락 요인으로 평가됩니다. 자기소개서의 협업·커뮤니케이션 서술과 실제 답변 태도 간 괴리도 커 인성·커뮤니케이션 신뢰도가 낮게 책정되었습니다. 전반적으로 구조화된 답변 연습, 언어·표정 매너 교정, 문장력·용어 정확도 개선이 선행되어야 하며, 현재 종합 채용 권고는 “비추천(38/100점)”입니다.

핵심 인사이트
기술 경험은 비교적 탄탄하지만, “말하는 방식(구조·태도·언어)”이 치명적 약점으로 작동하고 있어, 면접 전 전면적인 커뮤니케이션 리빌딩이 필요합니다.

Section 02
포트폴리오 & 답변 통합 요약
문서·포트폴리오·답변을 한 번에 보는 성장 지도
▶

자기소개서와 포트폴리오는 학부 → 웹 FE → 모바일 RN → 프리랜서/사이드 프로젝트로 이어지는 커리어 흐름이 자연스럽고, LCP 개선·이탈률 감소·코드 중복 감소 등 수치 기반 성과가 잘 정리되어 있습니다. 협업·커뮤니케이션, 주간 데모, 제품 관점 등은 문서 전반에서 일관되게 드러나 강점으로 작용하지만, 지원 회사/직무에 맞춘 메시지가 부족해 “어디에나 복붙 가능한” 인상을 줍니다. 포트폴리오 프로젝트는 문제–행동–결과 구조와 본인 기여도, 기술 선택 이유를 조금만 보강하면 시니어 면접관에게도 설득력 있는 스토리텔링이 가능합니다. 답변에서는 구조화·요약 능력, 도메인/비즈니스 관점 언급이 부족해 문서에서 보여준 강점이 말하기 단계에서 충분히 전달되지 못하고 있습니다. 전반적으로 “문제 해결형 FE 개발자”라는 정체성은 좋지만, 이를 짧고 선명하게 요약해 회사 맥락과 연결하는 작업이 필요합니다.

강점: 수치 기반 성과 강점: 웹·모바일 풀 스택 경험 보완: 회사 맞춤 메시지 보완: 말하기 구조화

Section 03
면접 커뮤니케이션 개선 포인트
STAR/SPAR, 매너, 표정, 문장력 핵심만 정리
▶
  • 구조화된 답변: 모든 경험을 “상황–문제–행동–결과(STAR/SPAR)”로 말하는 연습이 최우선 과제입니다.
  • 언어·매너: 욕설·비속어는 어떤 상황에서도 금지하고, 질문 의도를 존중하는 태도를 모의 면접을 통해 체화해야 합니다.
  • 표정·비언어: 영상 촬영으로 기본 표정을 점검하고, 답변 시작·마무리 시 가벼운 미소와 고개 끄덕임을 의도적으로 넣어 부정 정서를 완화합니다.
  • 사례 중심 설명: React 마이그레이션, RN 로그인/푸시, 성능 개선 등 각 주제별로 “실제 한 장면”을 화면·코드·협업 관점에서 정리해 둡니다.
  • 문장력·용어 정확도: 답변 전 키워드 메모, 답변 녹음·필사를 통해 오타·비문·비전문 용어를 줄이고, 기술 용어를 정확히 사용합니다.
  • 핵심 메시지 요약: 각 경험에서 “무엇을 배웠는지”를 한 문장으로 정리하는 습관을 들여, 결론을 먼저 말하는 연습을 합니다.

Section 05
빠른 액션 플랜 (6–8개)
다음 4주 안에 실행할 수 있는 구체 행동 목록
▶
1) STAR/SPAR 답변 템플릿 작성
React 마이그레이션, RN 로그인/푸시, 성능 개선, 협업 사례 등 핵심 5개 경험을 STAR/SPAR 구조로 5줄 이내로 정리합니다.
2) 욕설·비속어 완전 차단 훈련
모의 면접 10문항을 녹화하며, 감정이 올라가는 질문에서도 욕설·비속어 없이 답변하는 연습을 반복합니다.
3) 자기소개서 “회사 맞춤” 문단 추가
지원하려는 서비스 유형(예: 구독, 커머스, 핀테크)에 맞춰 성과 사례 1개를 연결하는 3~4문장을 템플릿으로 만들어 둡니다.
4) 포트폴리오 프로젝트 구조 재정리
각 주요 프로젝트에 “문제–행동–결과 + 내 기여도(%) + 기술 선택 이유”를 짧은 블록으로 추가합니다.
5) GitHub README 강화
Habitly/To-Do/랜딩 페이지 등 2~3개 리포지토리에 소개, 스택, 주요 기능, 스크린샷, 배운 점 섹션을 추가합니다.
6) 성능 개선 전·후 지표 시각화
Lighthouse 리포트 캡처와 LCP/이탈률 전·후 그래프를 만들어 포트폴리오와 자기소개서에 링크 또는 이미지로 첨부합니다.
7) 모의 면접 스터디 참여
주 1회 온라인/오프라인 스터디를 찾아, 기술·경험 질문에 대한 구조화 답변과 피드백을 정기적으로 받습니다.
8) 기술 블로그 “Top 3” 정리
React SPA 마이그레이션, RN 안정화, FE 성능 체크리스트 등 2~3편을 선정해 포트폴리오 상단에 링크 섹션을 만듭니다.
